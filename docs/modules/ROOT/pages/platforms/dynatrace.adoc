= Dynatrace Integration

Full-stack monitoring of Redis Enterprise with Dynatrace.

== Overview

Dynatrace provides automated, AI-powered monitoring for Redis Enterprise deployments.

NOTE: The current recommended path is Dynatrace's published https://www.dynatrace.com/hub/detail/redis-enterprise-prometheus/[`Redis Enterprise - Prometheus` extension]. This repository still keeps a current ActiveGate bootstrap kickstarter in `dynatrace_v2/kickstarter/`, while the old self-managed extension assets live under `dynatrace_v2/legacy/`.

== Prerequisites

* Dynatrace account (SaaS or Managed)
* Dynatrace OneAgent or ActiveGate
* API token with the permissions required by the Dynatrace-published extension

== Installation

=== Using Dynatrace OneAgent

The OneAgent automatically discovers and monitors Redis Enterprise processes when installed on the same host.

. Install OneAgent on Redis Enterprise nodes
. OneAgent will automatically detect Redis processes
. Metrics will appear in Dynatrace within minutes

=== Using the Published Extension

For containerized or remote deployments:

. Configure Prometheus to scrape Redis Enterprise
. Install or activate the `Redis Enterprise - Prometheus` extension from Dynatrace Hub
. Configure the extension to target the Redis Enterprise endpoint on `https://<redis-endpoint>:8070/v2`

== Dashboard Configuration

Import the Redis Enterprise dashboard template:

. Navigate to Dashboards in Dynatrace
. Use the dashboards installed with the published extension, or import tenant-specific JSON if you maintain custom views
. Configure data source mapping as required in your tenant

== Alerting

Dynatrace provides AI-powered anomaly detection for Redis Enterprise metrics.

Recommended alert configurations:

* Memory usage anomalies
* Connection spikes
* Replication lag
* Database unavailability
